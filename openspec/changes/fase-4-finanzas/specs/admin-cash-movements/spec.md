# Admin Cash Movements Specification

## Purpose

Ledger of all cash movements (income and expense) with running balance. Movements are auto-generated by DB triggers or created manually.

## Requirements

### Requirement: Cash movement auto-generation from orders

When an order status changes to `confirmed`, the system SHALL automatically insert a cash_movement with type = 'income', amount = order.total, description = "Orden #{order_id}".

#### Scenario: Order confirmed creates income movement

- GIVEN an existing order (pending) for $50,000
- WHEN the admin changes status to "confirmed"
- THEN a cash_movement row is inserted with type=income, amount=50,000

### Requirement: Cash movement auto-generation from expenses

When a new expense row is inserted, the system SHALL automatically insert a cash_movement with type = 'expense', amount = expense.amount, reference_type = 'expense'.

#### Scenario: Expense created creates expense movement

- GIVEN an admin creates an expense for $15,000
- WHEN the insert succeeds
- THEN a cash_movement of type=expense for $15,000 is also created

### Requirement: Cash movement auto-generation from purchases

When a new purchase row is inserted, the system SHALL automatically insert a cash_movement with type = 'expense', amount = purchase.total, reference_type = 'purchase'.

#### Scenario: Purchase created creates expense movement

- GIVEN an admin creates a purchase for $90,000
- WHEN the purchase insert succeeds
- THEN a cash_movement of type=expense for $90,000 is created

### Requirement: Running balance calculation

The system SHALL compute current balance as `SUM(income) - SUM(expense)` from all cash_movements. The balance per movement SHALL be computed by ordering movements chronologically and applying running total.

#### Scenario: Running balance after three movements

- GIVEN movements in order: income $100k, expense $30k, income $50k
- WHEN querying the timeline with running balance
- THEN balances show: $100k, $70k, $120k

#### Scenario: No movements shows zero

- GIVEN an empty cash_movements table
- WHEN querying the balance summary
- THEN income=$0, expense=$0, balance=$0

### Requirement: Movement timeline pagination

The ledger SHALL display movements ordered by movement_date descending, paginated. Admin MAY filter by type (income/expense) and date range.

#### Scenario: Filter by expense type

- GIVEN 10 income and 10 expense movements
- WHEN the admin selects filter "expense"
- THEN only expense movements are displayed
